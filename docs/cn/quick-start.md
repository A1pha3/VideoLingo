# 快速开始

> 5 分钟体验 VideoLingo 完整视频翻译流程

## 学习目标

完成本教程后，你将能够：

- 启动 VideoLingo Web 界面
- 完成一个视频的翻译和字幕生成
- 了解基本的工作流程

## 前置要求

在开始之前，请确保你的系统满足以下条件：

| 要求 | 说明 | 检查命令 |
|------|------|----------|
| Python | 3.10 版本 | `python --version` |
| FFmpeg | 已安装并可访问 | `ffmpeg -version` |
| GPU（可选） | NVIDIA GPU + CUDA 12.4+ | `nvidia-smi` |

> 💡 **提示**：FFmpeg 是必需的。如果未安装，运行 `python install.py` 时会得到安装指导。

> 🔍 **为什么需要 GPU？** WhisperX 转录在有 GPU 的情况下速度可提升 10-20 倍。没有 GPU 也可以使用云端 API 或 CPU 模式，只是速度较慢。

## 第一步：安装

```bash
# 克隆仓库
git clone https://github.com/Huanshere/VideoLingo.git
cd VideoLingo

# 运行安装脚本
python install.py
```

安装脚本会：

1. 检测你的 GPU 并选择合适的 PyTorch 版本
2. 安装所有 Python 依赖
3. 验证 FFmpeg 安装
4. 自动启动 Web 界面

> ⚠️ **注意**：首次安装需要下载模型文件（约 3GB），请确保网络畅通。

## 第二步：启动 Web 界面

```bash
streamlit run st.py
```

启动成功后，浏览器会自动打开 `http://localhost:8501`

> 🔍 **为什么使用 Streamlit？** Streamlit 提供了直观的 Web 界面，无需命令行操作即可完成复杂的视频处理流程。

## 第三步：下载视频

在 Web 界面的 **a. 下载视频** 区域：

1. 选择 **YouTube 下载** 标签
2. 输入 YouTube 视频链接
3. 点击 **下载视频**
4. 等待下载完成（进度会实时显示）

> 🎯 **替代方案**：你也可以上传本地视频文件（支持 mp4、mov、avi、mkv 等格式）

## 第四步：生成字幕

在 **b. 翻译和生成字幕** 区域：

1. 在左侧设置面板配置：
   - **目标语言**：选择翻译目标语言
   - **API 设置**：输入 LLM API 密钥
   - **Whisper 设置**：选择运行模式（本地/云端）

2. 点击 **开始处理字幕**

3. 等待处理完成（约 3-10 分钟，取决于视频长度）

处理流程包括：

- 🎙️ WhisperX 词级转录
- ✂️ NLP + LLM 句子分割
- 📝 视频摘要和术语提取
- 🔄 三步翻译（忠实→反思→改编）
- ⚡ 字幕时间轴对齐

> 🔍 **为什么是三步翻译？** 忠实翻译确保准确性，反思翻译发现问题，改编翻译优化表达。这种流程比单次翻译质量提升 30% 以上。

4. 完成后可以：
   - 预览带字幕的视频
   - 下载所有 SRT 字幕文件
   - 归档到历史记录

## 第五步：配音（可选）

如果需要生成配音，在 **c. 配音** 区域：

1. 在左侧设置面板配置 TTS：
   - **TTS 方法**：选择配音引擎（推荐 Azure TTS）
   - **语音选择**：选择配音声音
   - **API 密钥**：输入对应的 API 密钥

2. 点击 **开始音频处理**

3. 等待配音完成（约 5-15 分钟）

配音流程包括：

- 🔊 生成音频任务
- 🎵 提取参考音频
- 🗣️ 生成 TTS 音频
- 🔄 合并完整音轨
- 🎞️ 合成最终视频

## 输出文件

处理完成后，在 `output/` 目录会生成：

```
output/
├── output_sub.mp4         # 带字幕的视频
├── output_dub.mp4         # 带配音的视频（如果进行了配音）
├── log/
│   ├── terminology.json   # 提取的术语表
│   └── gpt_log/           # LLM 调用日志
└── srt_files/
    ├── zh-CN.srt          # 中文字幕
    ├── en.srt             # 英文字幕
    └── bilingual.srt      # 双语字幕
```

## 自测问题

完成教程后，尝试回答以下问题：

1. **WhisperX 的作用是什么？为什么需要词级对齐？**
   
   <details>
   <summary>点击查看答案</summary>
   WhisperX 负责语音识别和转录。词级对齐可以精确到每个单词的开始和结束时间，这对于后续的字幕分割和配音同步至关重要。
   </details>

2. **三步翻译流程中，每一步的作用是什么？**
   
   <details>
   <summary>点击查看答案</summary>
   - 忠实翻译：准确传达原文含义，不添加不删减
   - 反思翻译：分析直译的问题，发现表达不当之处
   - 改编翻译：优化语言流畅度，适配目标语言文化
   </details>

3. **Demucs 人声分离在什么情况下需要启用？**
   
   <details>
   <summary>点击查看答案</summary>
   当视频有明显的背景音乐或噪音时，启用 Demucs 可以分离人声，显著提高 WhisperX 的识别准确率。
   </details>

## 常见问题

### Q: 处理需要多长时间？

| 视频时长 | 字幕处理 | 配音处理 |
|----------|----------|----------|
| 5 分钟 | 约 3-5 分钟 | 约 5-8 分钟 |
| 10 分钟 | 约 5-10 分钟 | 约 10-15 分钟 |
| 30 分钟 | 约 15-20 分钟 | 约 30-40 分钟 |

### Q: 如何处理有背景音乐的视频？

在设置中启用 **Demucs 人声分离**，可以在转录前分离人声和背景音乐，显著提高识别准确率。

### Q: 可以批量处理多个视频吗？

可以。使用批处理模式：

```bash
# 1. 在 batch/tasks_setting.xlsx 中配置任务
# 2. 将视频放入 batch/input/ 目录
# 3. 运行批处理
python -m batch.utils.batch_processor
```

### Q: 没有 GPU 怎么办？

三种解决方案：

1. **使用云端 WhisperX**：在配置中选择 `runtime: cloud`
2. **使用 CPU 模式**：速度较慢但可用
3. **使用 Colab**：在 Google Colab 中运行，免费使用 GPU
